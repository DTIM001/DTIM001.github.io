20220918DAS支撑_瀑布图出现“栅栏”

2022年9月18日，健家次对瀑布图的出现“栅栏”的情况提出问题，“显示器分辨率不合适”的解释显然不能解决他们的疑惑。本篇笔记仅能陈述问题，并从单道的尺度证明数据情况正常，暂时没有该问题具体的解释思路。

问题陈述

  • DAS的瀑布图界面在长时间工作后会出现“栅栏”形式的图样，明显不是正常信号。青藏所那边想知道造成这种异常的原因，以及是否会对信号本身造成影响。

我的直接印象是：不会对信号造成影响，只不过是显示错误。
老板的解释是：不会对信号造成影响，是由显示器造成的。

回放检查

使用青藏所反馈的实例文件：”0916200733-00282.tdms”

  • 回放图像中没有明显栅格出现，因为回放图像的时间尺标为20s，而实时图像的时间尺标为0.5s。从实时图像上栅格出现的周期进行估计，其频率约为50Hz左右。

频谱检查

  • 对各道的频谱进行逐一查看，发现在55Hz左右存在尖刺，下面展示112道和200道德频谱，并通过扩展200道频谱的尺标，定位尖刺频率存在于57Hz左右，与民用交流电频率相符。

总结

  • 从本次检查结果来看，市电的民用频率会对DAS的数据造成影响：在瀑布图内出现“栅栏”。

  • 徐老师关于屏幕刷新的判断并没有进行验证，我也没有想好该怎么办。（但我猜测应该不是这个原因）

  • 确切的原因是什么？以及如何避免？

    • 电源净化器
    • 等到设备拿到手后需要在实验室情况下看看接市电以及“电小二”便携电池是否也存在50Hz左右的干扰。
    • 可以对比一下海试时的数据，可以找深海所他们确认一下当时的供电设计。

选择 Hexo+github 方案进行个人博客构建。本篇笔记记录搭建的方案与细节，如下图所示：

1. 本地Git配置

  • 安装Git,并查看是否安装成功。
  • 安装成功后配置git连接的账户
  • 生成与 Github 连接的ssh认证密钥
  • 密钥生成后会自动保存在 C:\Users\_admin_name_\.ssh\id_rsa.pub 中，可使用记
    事本打开。

2. Github配置

  • 确定github用户名（本文用admin指代）。创建新库并命名为：”admin.github.io”。（下图标红，是因为我以存在同名库）

  • 配置布设密钥。进到新建的Github仓库 点击右上角setting进入设置找到Deploy keys, 选择Add Deploy keys ,”Title” 随便填写, “Key” 的内容填刚才id_rsa.pub文件中复制的内容

注意：库的名称一定要保证格式为“github用户名.github.io”。

  • 添加完成后，进行测试
  • 你将会看到

3. Nodejs以及Hexo安装

  • Nodejs下载地址

  • 所有必备的应用程序安装完成后，即可使用 npm 安装 Hexo。（npm在安装Nodejs的过程中会附带下载）

4. 文章写作及发布

  • 在命令行模式（cmd、bash、shell）下进入想要将网页存放的路径，使用hexo init命令生成文件夹。

  • 通过记事本打开_config.yml，对页面配置进行修改

注：①网络教程中会将branch的设置为master，这会导致内容更新时网络页面不发生更新。要将branch设置为main；②repo地址为：https://github.com/username.github.io.git

  • 将路径切换到目标文件夹后，可执行新建文章、生成网页、本地伺服、服务器端发布等操作。
  • 完成布设后，在浏览器输入 username.github.io 即可打开发布的网页
